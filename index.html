Git e GitHub

Comandos Linux e Comandos GIT

O que é o Git

Git é um sistema de controle de versão distribuído usado para registrar o histórico de alterações em arquivos e projetos. Permite que múltiplos desenvolvedores trabalhem juntos no mesmo projeto.

O que é o GitHub

GitHub é uma plataforma de hospedagem de código-fonte baseada em Git. É um serviço web onde desenvolvedores podem armazenar projetos e colaborar com outros.

Diferenças entre Git e GitHub

Git é o software de controle de versão que você instala localmente.

GitHub é a plataforma online que hospeda repositórios Git.

Principais comandos Linux para Git Bash

ls - Listar arquivos e diretórios cd [diretório] - Navegar entre diretórios pwd - Mostrar diretório atual mkdir [nome] - Criar diretório rm [arquivo] - Remover arquivo

Principais comandos Git

git init - Inicializar repositório git add [arquivo] - Adicionar arquivo git commit -m "mensagem" - Criar commit git status - Verificar status git push - Enviar para repositório remoto git pull - Atualizar repositório local

Como criar um repositório no GitHub

  1. Faça login no GitHub
  2. Clique em "+" e selecione "New repository"
  3. Digite um nome para o repositório
  4. Escolha entre público ou privado
  5. Clique em "Create repository"

Como ligar um projeto existente ao GitHub

  1. Crie um novo repositório no GitHub
  2. No terminal: git init
  3. git add .
  4. git commit -m "primeiro commit"
  5. git branch -M main
  6. git remote add origin [URL_DO_REPOSITORIO]
  7. git push -u origin main

Como gerar uma versão com Git

  1. Faça alterações nos arquivos
  2. git add [arquivo] ou git add .
  3. git commit -m "Mensagem descritiva"

Como enviar para o GitHub

  1. Certifique-se de ter feito commit: git commit -m "mensagem"
  2. git push origin [nome-da-branch]
